硬件工具打包规范

定义RPM包依赖

需要在RPM包的SPEC文件里面指定硬件配置脚本所需的依赖关系，以Dell服务器为例：

  • RAID配置：依赖MegaCli
  • OOB配置：依赖ipmitool
  • BIOS配置：依赖syscfg

定义脚本路径

每个厂商定义的脚本路径如下：

# tree
.
|-- dell
|   |-- bios.sh
|   |-- oob.sh
|   `-- raid.sh
|-- hp
|   |-- bios.sh
|   |-- oob.sh
|   `-- raid.sh
`-- inspur
    |-- bios.sh
    |-- oob.sh
    `-- raid.sh

RPM包定义的脚本路径如下：

# rpm -ql dell-hw-tools
/opt/yunji/osinstall/dell/bios.sh
/opt/yunji/osinstall/dell/oob.sh
/opt/yunji/osinstall/dell/raid.sh

定义SPEC规范

以Dell为例，这里我们定义了一个spec打包文件：

# cat dell-hw-tools.spec
%define __spec_prep_post true
%define __spec_prep_pre true
%define __spec_build_post true
%define __spec_build_pre true
%define __spec_install_post true
%define __spec_install_pre true
%define __spec_clean_post true
%define __spec_clean_pre true
%define _binary_filedigest_algorithm 1
%define _build_binary_file_digest_algo 1
%define _binary_payload w9.gzdio

Name: dell-hw-tools
Version: 0.1
Release: 1
Summary: none
AutoReqProv: no
BuildRoot: %buildroot
Prefix: /opt/yunji/osinstall/dell
Group: default
License: GPLv3+
Vendor: CentOS
URL: none
Packager: admin@dell.com

Requires: MegaCli
Requires: ipmitool
Requires: syscfg

%description
none

%prep

%build

%install

%clean

%files
%defattr(-,root,root,-)
/opt/yunji/osinstall/dell/oob.sh
/opt/yunji/osinstall/dell/raid.sh
/opt/yunji/osinstall/dell/bios.sh

%changelog
